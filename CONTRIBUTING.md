# Contributing to Notes to My Future Self Skill

Thanks for your interest in improving this skill! Contributions are welcome and greatly appreciated.

## Ways to Contribute

### 1. Report Bugs or Issues
- Use the [Issues](https://github.com/bladngal/notes-to-future-self-skill/issues) section
- Check if the issue already exists before creating a new one
- Provide a clear description of the problem
- Include any error messages or steps to reproduce

### 2. Suggest Improvements
- Share ideas for new features or workflow improvements
- Open an Issue with the "enhancement" label
- Explain the use case and why you think it would be valuable

### 3. Improve Documentation
- Fix typos, unclear explanations, or outdated information
- Add examples or clarifications to existing guides
- Create templates for common use cases
- Suggest better organization or structure

### 4. Enhance the Script
- Improve the `organize_summary.py` helper script
- Add new command functionality
- Make the script more robust or user-friendly
- Fix bugs

### 5. Share Your Workflow
- Tell us how you're using this skill
- Share templates or organizational strategies that work for you
- Discuss what's working well and what could be better

## Getting Started

### For Documentation Improvements
1. Fork the repository
2. Create a branch for your changes (`git checkout -b improve/your-improvement`)
3. Make your changes
4. Submit a pull request with a clear description

### For Code Contributions
1. Fork the repository
2. Create a branch (`git checkout -b feature/new-feature` or `git checkout -b fix/bug-name`)
3. Make your changes and test them
4. Add or update relevant documentation
5. Submit a pull request

## Code Style and Standards

- Use Python 3.6+
- Follow PEP 8 style guidelines
- Add comments for complex logic
- Test your changes before submitting
- Update relevant documentation

## Pull Request Process

1. Ensure your changes don't break existing functionality
2. Update the CHANGELOG.md with your changes
3. Write a clear PR description explaining what and why
4. Be open to feedback and suggestions

## Questions or Discussions?

- Use the Issues section to start a discussion
- Ask questions about how to use the skill
- Share ideas for improvements
